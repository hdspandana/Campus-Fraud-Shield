# AI Disclosure

## Project Name  
**Campus Fraud Shield 🛡️**

---

## Purpose of AI Usage  
AI tools were used to assist in development, debugging, experimentation, and documentation of the project. All final design decisions, integration, and validation were performed by the team.

---

## AI Tools Used

### 1. Claude (Anthropic) – Primary Assistance Model  
Used for:
- Core system design suggestions  
- Improving fraud detection logic flow  
- Structuring backend pipeline and architecture ideas  
- Refining documentation and explanation clarity  

---

### 2. OpenAI ChatGPT  
Used for:
- Debugging Python / Streamlit issues  
- Helping with code structuring and refactoring  
- Assisting in README and PPT content preparation  
- Generating test cases and sample inputs  

---

### 3. Perplexity AI  
Used for:
- Quick research on fraud/scam detection approaches  
- Validating ML concepts and best practices  
- Cross-checking implementation strategies  

---

### 4. GitHub Copilot  
Used for:
- Real-time code suggestions during development  
- Debugging deployment and syntax-level issues  

---

### 5. Sentence Transformers  
Model used: `all-MiniLM-L6-v2`  
Used for:
- Semantic similarity between messages  
- Scam pattern matching and classification support  

---

### 6. FAISS (Facebook AI Similarity Search)  
Used for:
- Efficient vector similarity search  
- Matching user input with known scam patterns  

---

## Human Contribution

The team was responsible for:
- System architecture and workflow design  
- Integration of ML + backend + frontend components  
- Implementation of fraud detection logic  
- Dataset preparation and testing  
- UI development using Streamlit  
- End-to-end validation and debugging  
- Final documentation, PPT, and video demo preparation  

All AI-generated suggestions were carefully reviewed, modified, and tested before integration.

---

## Disclaimer

This project is developed strictly for educational and hackathon purposes.  
The system provides assistive fraud detection insights and should not be considered a fully authoritative or production-grade security system.
